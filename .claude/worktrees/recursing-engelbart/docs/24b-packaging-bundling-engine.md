# Packaging & Bundling Engine

Part of **Expansion Pack 2 — Phase B**. Generates value-stack, gateway-premium, complementary, and discount bundles from the offer catalog to drive upsell and average order value.

---

## Overview

The Bundling Engine evaluates the brand's active offer catalog, co-purchase frequency data, and market trends to produce 1–4 bundle recommendations per brand. Each bundle specifies which offers to combine, a discounted price point, an estimated upsell rate, and projected revenue impact. Results are persisted to `bundle_recommendations` and fully replaced on each recompute cycle.

---

## Inputs

| Input | Source | Description |
|---|---|---|
| `available_offers` | `offers` table | Active offers with `{id, name, price, features}`. |
| `customer_purchase_history` | Analytics pipeline | List of `{customer_id, purchased_offer_ids}` records. |
| `market_trends` | External / manual | List of `{trend_name, popular_bundles}` records. |
| `brand_name` | `brands.name` | Used in bundle naming. |
| `niche` | `brands.niche` | Used for fallback bundle naming. |

---

## Bundle Strategies

The engine evaluates four strategies in order. All triggered strategies produce a bundle row (up to 4 bundles max per brand). Deduplication is by sorted offer-ID combination — if two strategies produce the same offer set, only the first is retained.

### 1. Value Stack

- **Selection**: Top 2–3 offers by price (descending).
- **Discount**: 20% off combined price.
- **When**: ≥ 2 active offers exist.
- **Name pattern**: `"{brand_name} Value Stack"`.

### 2. Gateway Premium

- **Selection**: Cheapest offer + most expensive offer.
- **Discount**: 15% off combined price.
- **When**: ≥ 2 active offers and cheapest ≠ priciest.
- **Name pattern**: `"Gateway to {premium_offer_name}"`.

### 3. Complementary (Co-Purchase)

- **Selection**: Offer pairs that appear together in `customer_purchase_history`.
- **Discount**: 18% off combined price.
- **When**: Co-purchase data contains valid pairs matching active offers.
- **Name pattern**: `"{Offer A} + {Offer B} Combo"`.

### 4. Discount Only (Fallback)

- **Selection**: 2 cheapest offers.
- **Discount**: 25% off combined price.
- **When**: Fewer than 2 bundles generated by previous strategies.
- **Name pattern**: `"{niche} Essentials"`.

## Scoring Formulas

### Upsell Rate Estimate

```
upsell_rate = clamp(0.05 + discount × 0.30 + num_offers × 0.03, 0.0, 1.0)
```

### Revenue Impact Estimate

```
volume_estimate = max(10, int(50 × discount × num_offers))
revenue_impact = bundle_price × volume_estimate × upsell_rate
```

### Confidence Score

```
confidence = clamp(0.40 + num_offers × 0.08 + min(0.20, len(purchase_history) × 0.02))
```

## Outputs — `bundle_recommendations` Table

| Column | Type | Description |
|---|---|---|
| `id` | UUID PK | Row identifier. |
| `brand_id` | UUID FK | References `brands.id`. |
| `bundle_name` | String | Human-readable bundle name. |
| `offer_ids` | JSONB | Array of offer UUIDs included in the bundle. |
| `recommended_bundle_price` | Float | Discounted bundle price in USD. |
| `estimated_upsell_rate` | Float | Projected upsell conversion rate (0–1). |
| `estimated_revenue_impact` | Float | Projected monthly revenue impact. |
| `confidence` | Float | Model confidence (0–1). |
| `explanation` | Text | Human-readable rationale including strategy, savings %, and upsell rate. |
| `is_active` | Bool | Soft-delete flag. |
| `created_at` | Timestamp | Row creation time. |
| `updated_at` | Timestamp | Last recompute time. |

Unique constraint: `(brand_id, bundle_name)`.

---

## API Endpoints

| Method | Path | Role | Description |
|---|---|---|---|
| `GET` | `/brands/{id}/bundle-recommendations` | Any | All bundle recommendations for the brand, ordered by `estimated_revenue_impact` desc. |
| `POST` | `/brands/{id}/bundle-recommendations/recompute` | OPERATOR | Delete and rebuild all bundle recommendations for the brand. |

---

## Worker

| Task | Schedule | Queue |
|---|---|---|
| `recompute_all_bundle_recommendations` | Every 12 h (`:18`) | `revenue_ceiling` |

Iterates all active brands, deletes existing `bundle_recommendations`, and rebuilds 1–4 rows per brand.

---

## Frontend

Mounted at `/dashboard/expansion-pack2-b/bundling`. Displays a table with columns: Bundle Name, Offer IDs, Recommended Price, Upsell Rate, Revenue Impact, Confidence (progress bar), and Explanation. Recompute button (OPERATOR only) triggers the recompute endpoint and refreshes the table after 5 seconds.
